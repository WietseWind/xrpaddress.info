What & Why

The new 𝗫-address format can replace the use of a separate address and destination tag for XRP deposits. By using the 𝗫-address, it will no longer be possible to forget adding a destination tag or entering the wrong one when sending deposits to exchanges and other platforms accepting XRP deposits.

A new address!?

If you own an XRP account address (paper or hardware wallet), your address starts with 'r...' and doesn't require a destination tag for deposits. This will not change 🎉

When sending to an exchange, custodian wallet or payments platform, unlike when sending to your own XRP account address, you'll have to enter a numeric destination tag as well. That will change 🎉

The new 𝗫-address format will replace the combination of an 'r...' address in combination with a destination tag. By using one address that contains both the destination account and the destination tag, users can no longer make mistakes by making mistakes entering, or omitting the destination tag.

The new format

You can easily recognize the new address because it will always start with an upper case 𝗫. A new 𝗫-address will contain both the destination address and destination tag, and it looks like this: XV5sbjUmgPpvXv4ixFWZ5ptAYZ6PD2q1qM6owqNbug8W6KV.

It will take some time for all exchanges, clients, wallets, etc. to support the new address format. You will probably see the old format with separate numeric destination tags, and the new 𝗫-addresses both being offered. in the mean time.

Implementation

For developers and exchanges

If you operate a website receiving funds from users to one address using destination tags, or if you allow users to input an address for XRP/IOU withdrawals, you are very much invited to add 𝗫-address format support.

There's also a public API to encode / decode 𝗫-address formatted addresses: encode (/<address>/<tag>) and decode (/<x-address>). If you want to encode/decode in a browser, you can grab the browserified code (sample) from a CDN.

Apps sending XRP

When sending XRP to your users, your service most likely allows users to input their XRPL account address.

When users retrieve their deposit address from another exchange or custodial service, they may get an 𝗫-address formatted address.

For a good user experience, users should be able to enter both the 'r...' address and the new 𝗫-address format.

Apps receiving XRP

When showing the deposit address for your users, your service will currently show the 'r...' XRPL account address and the destination tag reserved for the user. The destination tag will be probably surrounded by warnings not to forget to add the tag when sending funds.

When adding the 𝗫-address formatted address, the user should be able to choose which address to display: the 𝗫-address formatted address without warnings and the 'r...' address and tag, with warnings.

Best practice

Everybody (users, exchanges, wallets) will benefit from a migration towards using the 𝗫-address formatted addresses, since users can no longer make mistakes when making deposits. No more uncertainty for users, no more manual funds recovery for exchanges and (custodian) wallet providers.

It should be really easy and clear for users that the 𝗫-address format is supported, and users should be encouraged to use the new address format.

When asking users to input a withdrawal address, please support both formats in one input. When an 𝗫-address is detected, remove the destination tag input. When detecting an 'r...' address and destination tag, inform the user they can ask the receiving exchange/platform to add 𝗫-address format support.

When giving users their deposit address, please give them 𝗫-address formatted address first. Then add a short explanation that they can also use the 'r...' format, and show this address and destination tag. Add a note to inform the user they can ask their sending app/exchange to add 𝗫-address format support.
